## 1.1 知识图谱 (Knowledge Graph)

### 什么是知识图谱？

知识图谱是一种**结构化的知识表示方式**，使用**节点(Node)**和**关系(Relationship)**来表示实体及其之间的联系。

```
传统数据存储：
┌─────────────────────────────────────────────┐
│ "客厅里有一台智能电视和一盏吊灯，            │
│  吊灯可以调节亮度和颜色..."                  │
└─────────────────────────────────────────────┘
        ↓ 转换为知识图谱 ↓

知识图谱表示：
┌─────────┐   CONTAINS   ┌─────────┐   HAS_CAPABILITY   ┌─────┐
│ 客厅    │─────────────→│ 吊灯    │──────────────────→│ 调光 │
│ (Room)  │              │(Device) │                    │(Cap) │
└─────────┘              └─────────┘                    └─────┘
     │                        │
     │ CONTAINS               │ HAS_CAPABILITY
     ↓                        ↓
┌─────────┐              ┌─────────┐
│ 智能电视 │              │ 变色    │
│(Device) │              │ (Cap)   │
└─────────┘              └─────────┘
```

### 知识图谱的核心要素

| 要素 | 英文 | 说明 | 示例 |
|------|------|------|------|
| 节点 | Node | 实体/概念 | 房间、设备、能力 |
| 标签 | Label | 节点类型 | :Room, :Device |
| 属性 | Property | 节点/关系的属性 | name: "客厅" |
| 关系 | Relationship | 节点间的连接 | CONTAINS, HAS |
| 方向 | Direction | 关系的方向性 | Room→Device |

### 为什么使用知识图谱？

```
┌─────────────────────────────────────────────────────────────────┐
│                    传统方法 vs 知识图谱                          │
├─────────────────────────────┬───────────────────────────────────┤
│        关系型数据库          │           知识图谱                │
├─────────────────────────────┼───────────────────────────────────┤
│ 多表JOIN查询复杂            │ 关系是一等公民，查询直观           │
│ 难以表达复杂关系            │ 天然支持多跳关系                   │
│ Schema变更成本高            │ Schema灵活可扩展                   │
│ 关系隐含在外键中            │ 关系显式且有语义                   │
└─────────────────────────────┴───────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐
│                        应用场景                                  │
├─────────────────┬─────────────────┬─────────────────────────────┤
│   🔍 搜索引擎    │   🏥 医疗健康    │      🏢 企业知识管理         │
│   Google知识卡片 │   疾病-症状-药物 │      组织架构、流程          │
├─────────────────┼─────────────────┼─────────────────────────────┤
│   🛒 电商推荐    │   🏠 智能家居    │      🤖 AI问答系统           │
│   商品关联推荐   │   设备-房间-场景 │      知识检索增强            │
```

## 1.2 图数据库 Neo4j

### 什么是Neo4j？

Neo4j是世界上最流行的**原生图数据库**，专门为存储和查询图结构数据而设计。

```
┌─────────────────────────────────────────────────────────────────┐
│                      Neo4j 技术架构                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    应用层 (Application)                  │   │
│  │   Python Driver │ JavaScript │ Java │ REST API          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              ▼                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    查询层 (Query)                        │   │
│  │              Cypher 查询语言解析与优化                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              ▼                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    存储层 (Storage)                      │   │
│  │         原生图存储引擎 (Native Graph Storage)            │   │
│  │    节点存储 │ 关系存储 │ 属性存储 │ 索引存储              │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Cypher查询语言

Cypher是Neo4j的声明式查询语言，语法接近自然语言，易于学习。

```cypher
-- 基本语法模式
-- 节点用圆括号：(n:Label {property: value})
-- 关系用方括号：-[r:TYPE]->

-- 示例1：创建节点
CREATE (room:Room {name: "客厅", floor: "一楼"})

-- 示例2：创建关系
MATCH (r:Room {name: "客厅"}), (d:Device {name: "智能电视"})
CREATE (r)-[:CONTAINS]->(d)

-- 示例3：查询模式
MATCH (r:Room)-[:CONTAINS]->(d:Device)-[:HAS_CAPABILITY]->(c:Capability)
WHERE r.name = "客厅"
RETURN r.name, d.name, collect(c.name) AS capabilities
```

### SQL vs Cypher 对比

```
┌─────────────────────────────────────────────────────────────────┐
│   查询：找出客厅中所有设备的功能                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  SQL (需要多次JOIN):                                            │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ SELECT r.name, d.name, c.name                           │   │
│  │ FROM rooms r                                            │   │
│  │ JOIN room_devices rd ON r.id = rd.room_id               │   │
│  │ JOIN devices d ON rd.device_id = d.id                   │   │
│  │ JOIN device_capabilities dc ON d.id = dc.device_id      │   │
│  │ JOIN capabilities c ON dc.capability_id = c.id          │   │
│  │ WHERE r.name = '客厅';                                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  Cypher (直接表达关系路径):                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ MATCH (r:Room {name: "客厅"})-[:CONTAINS]->(d:Device)   │   │
│  │       -[:HAS_CAPABILITY]->(c:Capability)                │   │
│  │ RETURN r.name, d.name, c.name                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 1.3 RAG与GraphRAG

### 什么是RAG？

**RAG (Retrieval-Augmented Generation)** 是一种将检索系统与生成模型结合的技术，让LLM能够基于外部知识回答问题。

```
┌─────────────────────────────────────────────────────────────────┐
│                      传统RAG工作流程                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  用户问题                                                        │
│      │                                                          │
│      ▼                                                          │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │   Embedding │───→│ 向量搜索    │───→│  Top-K文档   │         │
│  │   向量化    │    │ (相似度)    │    │   chunks    │         │
│  └─────────────┘    └─────────────┘    └─────────────┘         │
│                                              │                  │
│                                              ▼                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                        LLM                               │   │
│  │   System: 基于以下上下文回答问题                          │   │
│  │   Context: [检索到的文档片段]                            │   │
│  │   Question: [用户问题]                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              ▼                                  │
│                         生成回答                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### GraphRAG有何不同？

**GraphRAG** 使用**知识图谱**作为知识源，通过**图遍历**而非向量相似度来检索相关信息。

```
┌─────────────────────────────────────────────────────────────────┐
│                     GraphRAG工作流程                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  用户问题: "客厅有哪些可以调光的设备？"                           │
│      │                                                          │
│      ▼                                                          │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │  实体识别   │───→│ 图谱查询    │───→│  子图检索    │         │
│  │ "客厅"      │    │ Cypher     │    │  结构化结果  │         │
│  │ "调光"      │    │            │    │             │         │
│  └─────────────┘    └─────────────┘    └─────────────┘         │
│                                              │                  │
│                           ┌──────────────────┘                  │
│                           ▼                                     │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │              检索到的子图 (Subgraph)                      │   │
│  │                                                          │   │
│  │    ┌──────┐  CONTAINS  ┌──────┐  HAS_CAP  ┌──────┐      │   │
│  │    │ 客厅 │───────────→│ 吊灯 │──────────→│ 调光 │      │   │
│  │    └──────┘            └──────┘           └──────┘      │   │
│  │        │                                                 │   │
│  │        │ CONTAINS      ┌──────┐  HAS_CAP  ┌──────┐      │   │
│  │        └──────────────→│落地灯│──────────→│ 调光 │      │   │
│  │                        └──────┘           └──────┘      │   │
│  └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              ▼                                  │
│                      LLM生成回答                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 传统RAG vs GraphRAG 对比

```
┌─────────────────────────────────────────────────────────────────┐
│                    RAG vs GraphRAG 对比                          │
├──────────────────────┬──────────────────────────────────────────┤
│      传统RAG          │              GraphRAG                    │
├──────────────────────┼──────────────────────────────────────────┤
│ 知识源：文档chunks    │ 知识源：知识图谱                          │
├──────────────────────┼──────────────────────────────────────────┤
│ 检索方式：向量相似度  │ 检索方式：图遍历                          │
├──────────────────────┼──────────────────────────────────────────┤
│ 返回：文本片段        │ 返回：结构化子图                          │
├──────────────────────┼──────────────────────────────────────────┤
│ 关系：隐含在文本中    │ 关系：显式定义                            │
├──────────────────────┼──────────────────────────────────────────┤
│ 适合：开放域问答      │ 适合：结构化领域知识                      │
├──────────────────────┼──────────────────────────────────────────┤
│ 示例：                │ 示例：                                   │
│ "介绍一下机器学习"    │ "客厅里能调光的设备有哪些？"              │
└──────────────────────┴──────────────────────────────────────────┘
```

### 何时使用GraphRAG？

```
✅ 适合GraphRAG的场景：
   • 领域知识有明确的实体和关系
   • 需要多跳推理（A→B→C）
   • 需要精确的属性查询
   • 知识需要频繁更新

❌ 不适合GraphRAG的场景：
   • 开放域问答
   • 知识主要是非结构化文本
   • 没有明确的实体关系
```

## 1.4 LangChain框架

**LangChain**是一个用于构建LLM应用的开发框架，提供了模块化的组件来简化LLM应用开发。

```
┌─────────────────────────────────────────────────────────────────┐
│                    LangChain 核心组件                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                     Model I/O                             │  │
│  │  ┌─────────┐   ┌─────────────┐   ┌──────────────┐        │  │
│  │  │ Prompts │   │ LLM/Chat    │   │ Output       │        │  │
│  │  │ 提示模板 │   │ Models     │   │ Parsers      │        │  │
│  │  └─────────┘   └─────────────┘   └──────────────┘        │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                    Retrieval                              │  │
│  │  ┌─────────┐   ┌─────────────┐   ┌──────────────┐        │  │
│  │  │Document │   │ Text        │   │ Vector       │        │  │
│  │  │Loaders  │   │ Splitters   │   │ Stores       │        │  │
│  │  └─────────┘   └─────────────┘   └──────────────┘        │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                     Agents                                │  │
│  │  ┌─────────┐   ┌─────────────┐   ┌──────────────┐        │  │
│  │  │ Tools   │   │ Agent       │   │ Memory       │        │  │
│  │  │ 工具    │   │ Executors   │   │ 记忆         │        │  │
│  │  └─────────┘   └─────────────┘   └──────────────┘        │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### LangChain核心概念

| 概念 | 说明 | 本项目中的应用 |
|------|------|----------------|
| **Prompt Template** | 结构化的提示模板 | 意图解析、计划生成提示 |
| **Chat Model** | 对话模型封装 | OpenAI GPT-4o-mini |
| **Output Parser** | 输出解析器 | JSON解析为Python对象 |
| **Chain** | 组件链式调用 | Prompt → LLM → Parser |
| **Tool** | 可调用的外部工具 | Neo4j查询工具 |

### 代码示例

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

# 1. 定义Prompt模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个智能家居助手。根据用户请求提取意图。"),
    ("human", "用户说：{user_input}")
])

# 2. 创建LLM实例
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# 3. 组合成Chain
chain = prompt | llm

# 4. 调用
result = chain.invoke({"user_input": "把客厅灯调暗一点"})
```

### LangGraph是什么？

**LangGraph**是LangChain团队开发的用于构建**有状态多步骤Agent**的框架。它将Agent建模为**状态机 (State Machine)**。

```
┌─────────────────────────────────────────────────────────────────┐
│                    LangGraph 核心概念                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                    State (状态)                          │  │
│   │                                                          │  │
│   │  AgentState = {                                          │  │
│   │      user_input: str,        # 用户输入                  │  │
│   │      parsed_intent: dict,    # 解析后的意图               │  │
│   │      context: str,           # 检索到的上下文             │  │
│   │      action_plan: list,      # 行动计划                  │  │
│   │      response: str           # 最终响应                  │  │
│   │  }                                                       │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                    Nodes (节点)                          │  │
│   │                                                          │  │
│   │  每个节点是一个处理函数：                                  │  │
│   │  def node_function(state: AgentState) -> dict:           │  │
│   │      # 读取状态，处理，返回更新                           │  │
│   │      return {"field": new_value}                         │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                    Edges (边)                            │  │
│   │                                                          │  │
│   │  • 普通边：A → B (总是执行)                               │  │
│   │  • 条件边：A → B or C (根据条件选择)                      │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
