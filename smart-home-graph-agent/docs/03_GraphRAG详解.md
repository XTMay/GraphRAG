## 3.1 GraphRAG原理深入

### 传统RAG的局限性

```
┌─────────────────────────────────────────────────────────────────┐
│                   传统RAG的问题                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  问题1: 语义相似 ≠ 相关                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 查询: "客厅的智能电视"                                   │   │
│  │                                                          │   │
│  │ 向量相似结果:                                            │   │
│  │   ✗ "智能电视的维修方法" (高相似度，但无关)              │   │
│  │   ✗ "客厅装修风格" (高相似度，但无关)                    │   │
│  │   △ "客厅有一台智能电视" (相关，但可能排在后面)          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  问题2: 无法捕捉结构化关系                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 查询: "哪些设备可以调光？"                               │   │
│  │                                                          │   │
│  │ 传统RAG: 只能找到文本中提到"调光"的文档                  │   │
│  │ 无法利用: Device -[HAS_CAPABILITY]-> "dim" 这样的结构   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  问题3: 上下文碎片化                                            │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 文档被切分为chunks后，关联信息可能分散在不同chunk        │   │
│  │                                                          │   │
│  │ Chunk 1: "客厅有一台智能电视..."                        │   │
│  │ Chunk 2: "智能电视可以调节音量..."                      │   │
│  │ Chunk 3: "电视支持HDMI输入..."                          │   │
│  │                                                          │   │
│  │ 如果只检索到Chunk 1，会丢失电视的能力信息                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### GraphRAG如何解决这些问题

```
┌─────────────────────────────────────────────────────────────────┐
│                   GraphRAG的优势                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  解决方案1: 精确的实体匹配                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 查询: "客厅的智能电视"                                   │   │
│  │                                                          │   │
│  │ GraphRAG:                                                │   │
│  │   1. 识别实体: "客厅" (Room), "智能电视" (Device)        │   │
│  │   2. 精确查询: MATCH (r:Room {name:"客厅"})              │   │
│  │                -[:CONTAINS]->(d:Device {name:"智能电视"})│   │
│  │   3. 返回确切匹配，无噪声                                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  解决方案2: 关系遍历                                            │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 查询: "哪些设备可以调光？"                               │   │
│  │                                                          │   │
│  │ GraphRAG:                                                │   │
│  │   MATCH (d:Device)-[:HAS_CAPABILITY]->(c:Capability)     │   │
│  │   WHERE c.name = "dim"                                   │   │
│  │   RETURN d.name                                          │   │
│  │                                                          │   │
│  │   结果: 吊灯, 落地灯, 厨房灯, 卧室灯, 床头灯, 台灯       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  解决方案3: 子图检索保持完整性                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 查询"客厅"时，一次性获取:                                │   │
│  │                                                          │   │
│  │   客厅 ────CONTAINS───→ 智能电视 ──HAS_CAP──→ power     │   │
│  │     │                      │                   volume    │   │
│  │     │                      └──HAS_CAP──→ input_select    │   │
│  │     │                                                    │   │
│  │     ├──CONTAINS───→ 吊灯 ──HAS_CAP──→ power, dim, color │   │
│  │     │                                                    │   │
│  │     └──CONTAINS───→ ... (其他设备)                       │   │
│  │                                                          │   │
│  │   完整的上下文，无遗漏                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### GraphRAG工作流程详解

```
┌─────────────────────────────────────────────────────────────────┐
│                   GraphRAG完整工作流程                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  用户输入: "把客厅弄得舒适一点"                                  │
│       │                                                         │
│       ▼                                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Step 1: 意图解析 (Intent Parsing)                       │   │
│  │                                                          │   │
│  │  使用LLM提取:                                            │   │
│  │  • 目标实体: "客厅" → Room                               │   │
│  │  • 动作意图: "舒适" → mood: cozy                         │   │
│  │  • 隐含场景: 可能是 relaxed 类场景                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│       │                                                         │
│       ▼                                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Step 2: 策略选择 (Strategy Selection)                   │   │
│  │                                                          │   │
│  │  根据意图选择检索策略:                                    │   │
│  │  • 提到房间 → ROOM_CONTEXT                               │   │
│  │  • 提到动作 → CAPABILITY_SEARCH                          │   │
│  │  • 提到场景 → SCENE_LOOKUP                               │   │
│  │  • 模糊请求 → KEYWORD_SEARCH                             │   │
│  │                                                          │   │
│  │  本例: 使用 ROOM_CONTEXT + 可能的 SCENE_LOOKUP           │   │
│  └─────────────────────────────────────────────────────────┘   │
│       │                                                         │
│       ▼                                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Step 3: 查询生成 (Query Generation)                     │   │
│  │                                                          │   │
│  │  生成Cypher查询:                                         │   │
│  │                                                          │   │
│  │  // 查询1: 房间上下文                                    │   │
│  │  MATCH (r:Room)-[:CONTAINS]->(d:Device)                  │   │
│  │  WHERE r.name CONTAINS "客厅"                            │   │
│  │  OPTIONAL MATCH (d)-[:HAS_CAPABILITY]->(c:Capability)    │   │
│  │  ...                                                     │   │
│  │                                                          │   │
│  │  // 查询2: 相关场景                                      │   │
│  │  MATCH (s:Scene)-[:APPLIES_TO]->(r:Room)                 │   │
│  │  WHERE r.name CONTAINS "客厅" AND s.mood = "relaxed"     │   │
│  │  ...                                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│       │                                                         │
│       ▼                                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Step 4: 查询执行 (Query Execution)                      │   │
│  │                                                          │   │
│  │  Neo4j返回结构化结果:                                    │   │
│  │  {                                                       │   │
│  │    "room": {"name": "Living Room", "type": "entertainment"},│
│  │    "devices": [                                          │   │
│  │      {"name": "Smart TV", "capabilities": [...]},        │   │
│  │      {"name": "Ceiling Light", "capabilities": [...]},   │   │
│  │      ...                                                 │   │
│  │    ],                                                    │   │
│  │    "scenes": [{"name": "Movie Night", ...}]              │   │
│  │  }                                                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│       │                                                         │
│       ▼                                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Step 5: 上下文格式化 (Context Formatting)               │   │
│  │                                                          │   │
│  │  将结构化数据转为LLM友好的文本:                           │   │
│  │                                                          │   │
│  │  # 房间: 客厅 (entertainment)                            │   │
│  │  ## 设备                                                 │   │
│  │  - 智能电视: power, volume, input_select                 │   │
│  │  - 吊灯: power, dim, color                               │   │
│  │  - 落地灯: power, dim                                    │   │
│  │  ...                                                     │   │
│  │  ## 相关场景                                             │   │
│  │  - Movie Night: 灯光调暗, 关窗帘, 开电视                  │   │
│  └─────────────────────────────────────────────────────────┘   │
│       │                                                         │
│       ▼                                                         │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  Step 6: LLM推理 (LLM Reasoning)                         │   │
│  │                                                          │   │
│  │  System: 你是智能家居助手...                             │   │
│  │  Context: [格式化后的上下文]                             │   │
│  │  User: 把客厅弄得舒适一点                                │   │
│  │                                                          │   │
│  │  LLM输出:                                                │   │
│  │  {                                                       │   │
│  │    "reasoning": "为了创造舒适氛围，调暗灯光...",          │   │
│  │    "actions": [                                          │   │
│  │      {"device": "吊灯", "action": "dim", "value": 30},   │   │
│  │      {"device": "落地灯", "action": "dim", "value": 40}  │   │
│  │    ]                                                     │   │
│  │  }                                                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 策略1: ROOM_CONTEXT (房间上下文)

```
┌─────────────────────────────────────────────────────────────────┐
│              策略1: ROOM_CONTEXT (房间上下文)                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  📋 使用场景                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ • "打开客厅的灯"                                        │   │
│  │ • "把卧室弄得舒服一点"                                   │   │
│  │ • "客厅有哪些设备？"                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  📊 检索内容                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                          │   │
│  │  ┌──────────────────────────────────────────────────┐   │   │
│  │  │                     Room                          │   │   │
│  │  │ • name, type, floor                               │   │   │
│  │  └───────────────────────┬──────────────────────────┘   │   │
│  │                          │                               │   │
│  │          ┌───────────────┼───────────────┐               │   │
│  │          ▼               ▼               ▼               │   │
│  │  ┌───────────┐   ┌───────────┐   ┌───────────┐          │   │
│  │  │  Device 1 │   │  Device 2 │   │  Device N │          │   │
│  │  │ +能力列表  │   │ +能力列表  │   │ +能力列表  │          │   │
│  │  └───────────┘   └───────────┘   └───────────┘          │   │
│  │                                                          │   │
│  │  ┌───────────────────────────────────────────────────┐  │   │
│  │  │ 相关场景 (适用于该房间的Scene)                     │  │   │
│  │  └───────────────────────────────────────────────────┘  │   │
│  │                                                          │   │
│  │  ┌───────────────────────────────────────────────────┐  │   │
│  │  │ 相邻房间 (通过ADJACENT_TO关系)                     │  │   │
│  │  └───────────────────────────────────────────────────┘  │   │
│  │                                                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  🔍 Cypher查询                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ MATCH (r:Room)-[:CONTAINS]->(d:Device)                   │   │
│  │ WHERE toLower(r.name) CONTAINS toLower($room_name)       │   │
│  │ OPTIONAL MATCH (d)-[:HAS_CAPABILITY]->(c:Capability)     │   │
│  │ WITH r, d, collect(c) AS capabilities                    │   │
│  │ WITH r, collect({device: d, caps: capabilities}) AS devs │   │
│  │ OPTIONAL MATCH (s:Scene)-[:APPLIES_TO]->(r)              │   │
│  │ OPTIONAL MATCH (r)-[:ADJACENT_TO]->(adj:Room)            │   │
│  │ RETURN r, devs, collect(s) AS scenes, collect(adj) AS adj│   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  📝 格式化输出示例                                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ # 房间: 客厅 (entertainment, 一楼)                       │   │
│  │ ## 设备 (6个)                                            │   │
│  │ - **智能电视** (television) ✓                            │   │
│  │   - 能力: power, volume, input_select                    │   │
│  │ - **吊灯** (light) ✓                                     │   │
│  │   - 能力: power, dim, color                              │   │
│  │ ...                                                      │   │
│  │ ## 相关场景                                              │   │
│  │ - Movie Night: 灯光调暗，关窗帘                          │   │
│  │ - Party Mode: 彩色灯光，音乐                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 策略2: CAPABILITY_SEARCH (能力搜索)

```
┌─────────────────────────────────────────────────────────────────┐
│            策略2: CAPABILITY_SEARCH (能力搜索)                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  📋 使用场景                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ • "调暗所有灯" (能力: dim)                               │   │
│  │ • "播放音乐" (能力: play_music)                          │   │
│  │ • "调高温度" (能力: temperature)                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  📊 检索逻辑                                                    │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                          │   │
│  │       输入: ["dim"]                                      │   │
│  │             │                                            │   │
│  │             ▼                                            │   │
│  │  ┌─────────────────────────────────────────────────┐    │   │
│  │  │         查找所有具有该能力的设备                  │    │   │
│  │  │                                                  │    │   │
│  │  │  Capability ←──HAS_CAPABILITY── Device           │    │   │
│  │  │    "dim"                         │               │    │   │
│  │  │                                  ▼               │    │   │
│  │  │                          ┌──────────────┐        │    │   │
│  │  │                          │  返回设备列表 │        │    │   │
│  │  │                          │  按房间分组   │        │    │   │
│  │  │                          └──────────────┘        │    │   │
│  │  └─────────────────────────────────────────────────┘    │   │
│  │                                                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  🔍 Cypher查询                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ MATCH (r:Room)-[:CONTAINS]->(d:Device)                   │   │
│  │       -[:HAS_CAPABILITY]->(c:Capability)                 │   │

│  │        d.device_id, d.name, d.device_type,               │   │
│  │        collect(c) AS matching_capabilities               │   │
│  │ ORDER BY r.name                                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  📝 格式化输出示例                                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ # 具有 [dim] 能力的设备                                  │   │
│  │                                                          │   │
│  │ ## 客厅                                                  │   │

│  │ ## 厨房                                                  │   │
│  │ - **厨房灯** (light)                                     │   │

│  │ ...                                                      │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │

```

### 策略3: SCENE_LOOKUP (场景查找)

## 3.3 上下文格式化

### 为什么格式化很重要？

```
┌─────────────────────────────────────────────────────────────────┐
│                   格式化的重要性                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ❌ 不好的格式 (原始JSON)                                       │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ {"room":{"name":"Living Room","type":"entertainment"},   │   │
│  │ "devices":[{"device_id":"living_ceiling_01","name":     │   │
│  │ "Ceiling Light","device_type":"light","capabilities":   │   │
│  │ [{"name":"power"},{"name":"dim"},{"name":"color"}]}...  │   │
│  │                                                          │   │
│  │ 问题:                                                    │   │
│  │ • 难以阅读                                               │   │
│  │ • 消耗更多token                                          │   │
│  │ • LLM可能漏掉关键信息                                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ✅ 好的格式 (结构化Markdown)                                   │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ # Room: Living Room (entertainment, Ground Floor)        │   │
│  │                                                          │   │
│  │ ## Devices (6 total)                                     │   │
│  │ - **Ceiling Light** (light) ✓                            │   │
│  │   - Capabilities: power, dim, color                      │   │
│  │ - **Smart TV** (television) ✓                            │   │
│  │   - Capabilities: power, volume, input_select            │   │
│  │                                                          │   │
│  │ 优点:                                                    │   │
│  │ • 层次清晰                                               │   │
│  │ • 关键信息突出                                           │   │
│  │ • Token效率高                                            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### SmartHomeRetriever类结构

```python
class SmartHomeRetriever:
    """
    GraphRAG检索器

    核心方法:
    - get_room_context()      → ROOM_CONTEXT策略
    - get_devices_by_capability() → CAPABILITY_SEARCH策略
    - get_scene_context()     → SCENE_LOOKUP策略
    - search_by_keywords()    → KEYWORD_SEARCH策略
    - get_multi_room_context() → MULTI_ROOM策略

    内部方法:
    - _format_room_context()  → 格式化房间上下文
    - _format_capability_context() → 格式化能力搜索结果
    - _format_scene_context() → 格式化场景信息
    """
```
